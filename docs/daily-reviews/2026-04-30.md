- [ ] **[MINOR]** `examples/product-demo/test/stores/domain/useAgentSession.test.ts:415` — two-microtask flush hard-codes internal `await` depth of `setCognitionMode` <!-- f:5ce68c3.1 -->
  <details><summary>details</summary>

  **Problem:** Test timing relies on exactly two `await Promise.resolve()` calls to advance `setCognitionMode` past `probe()` and onto the `construct()` suspension

  **Why it matters:** If `setCognitionMode` gains one more `await` before reaching `construct()` — say, an extra guard or log step — the race setup silently fails: `init()` lands before `construct()` is awaited, the stale-epoch guard triggers at the probe-stale check instead, and `disposeReasoner` is never reached with the augmented reasoner, causing `expect(disposed).toBe(true)` to fail for a non-obvious reason

  **Fix:**

  ```diff
  -    await Promise.resolve();
  -    await Promise.resolve();
  +    await flushPromises(); // import from '@vue/test-utils' if not already present
  ```

  Apply to both the dispose test (~line 415) and the no-dispose test (~line 449). `flushPromises()` drains all settled microtasks while leaving the pending `construct()` promise unresolved (it only resolves when `resolveConstruct(...)` is called explicitly), so the race setup remains valid.

  </details>
