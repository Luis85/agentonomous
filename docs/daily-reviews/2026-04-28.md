- [ ] **[MINOR]** `examples/product-demo/src/stores/domain/useAgentSession.ts:321` — stale-epoch guard abandons freshly-constructed Reasoner without disposal <!-- f:98c191b.1 -->
  <details><summary>details</summary>

  **Problem:** After `await mode.construct()` resolves, if the stale-completion check fires (`myToken !== cognitionToken || agent.value !== targetAgent`), the function returns immediately, leaking the newly-constructed `Reasoner`

  **Why it matters:** A `TfjsReasoner` (learning mode) holds WebGL allocations; abandoning it without calling `dispose()` leaks GPU memory. The analogous code path in `cognitionSwitcher.ts` (lines 461–464 and 722–726 in the refactored diff) correctly calls `disposeIfOwned(reasoner)` on its stale-epoch check — the store path is inconsistent.

  **Fix:**

  ```diff
   const reasoner = await mode.construct();
  -if (myToken !== cognitionToken || agent.value !== targetAgent) return;
  +if (myToken !== cognitionToken || agent.value !== targetAgent) {
  +  if ('dispose' in reasoner && typeof (reasoner as { dispose?: unknown }).dispose === 'function') {
  +    (reasoner as { dispose: () => void }).dispose();
  +  }
  +  return;
  +}
   targetAgent.setReasoner(reasoner);
  ```

  </details>

  <details><summary>details</summary>

  **Problem:** `setCognitionMode` calls `targetAgent.setReasoner(reasoner)` but never calls `setLearner`, so a user who selects `learning` mode via the HUD picker (when TF.js is installed) gets a `TfjsReasoner` doing inference with no `TfjsLearner` accumulating outcomes — online batch training is silently disabled

  -// legacy cognitionSwitcher with its loss sparkline + prediction strip.
  +// Pillar-1 slice 1.3: minimal placeholder. 'learning' mode is intentionally
  +// disabled here — the learner-wiring path lives in cognitionSwitcher (Pillar-2).
   const cognitionAvailable = ref<Record<CognitionModeSpec['id'], boolean>>({
