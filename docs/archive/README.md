# Archive

Shipped or superseded planning docs.

- `plans/` — implementation plans whose roadmap rows have all landed (or whose
  goals were folded into a successor plan). Kept for historical context.
- `specs/` — design / brainstorm docs whose outcomes are now reflected in the
  code or in a successor spec.

Files keep their original `YYYY-MM-DD-<slug>.md` name so `git mv` preserves
history. Do not delete — Codex review history and earlier PR descriptions link
back to these paths.
